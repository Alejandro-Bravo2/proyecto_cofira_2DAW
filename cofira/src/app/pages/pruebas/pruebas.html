Demostración de SVGs Optimizados

Esta página demuestra el proceso de optimización de iconos SVG usando SVGO. Los iconos se han extraído del código inline y guardado como archivos .svg independientes para facilitar su reutilización y optimización.

Galería de Iconos SVG

Todos estos iconos están cargados desde archivos .svg externos ubicados en /assets/icons_optimizados/. Se cargan usando la etiqueta <img> con el atributo loading="lazy".

    • }

Resultados de Optimización

Los SVGs fueron procesados usando SVGOMG, la versión web de SVGO. Los iconos originales provenían de Heroicons, una librería que ya genera SVGs muy optimizados, por lo que la reducción adicional es mínima.

Nota: En proyectos con SVGs de herramientas como Illustrator o Figma, la reducción puede ser del 40-60% ya que estos suelen incluir metadatos innecesarios.

Ejemplo de Uso

Así es como se implementan los iconos SVG optimizados en el HTML: 

<!-- Carga de SVG como imagen con lazy loading -->
<img
  src="/assets/icons_optimizados/check-circle_opt.svg"
  alt="Icono de verificación"
  loading="lazy"
  width="24"
  height="24">

<!-- Dentro de Angular con binding dinámico -->
<img
  [src]="'/assets/icons_optimizados/' + iconName + '.svg'"
  [alt]="iconDescription"
  loading="lazy"
  width="24"
  height="24">

Nota sobre SVGs inline vs externos

En el resto de la aplicación, los iconos se mantienen como SVG inline porque utilizan currentColor para heredar el color del texto CSS padre. Esto permite que los iconos cambien de color automáticamente según el contexto (modo oscuro, estados hover, etc.).

Los SVGs externos mostrados aquí demuestran el conocimiento de la técnica de optimización y extracción, pero no se aplican globalmente para no perder la funcionalidad de colores dinámicos.
